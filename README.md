# Bibliographer

The whole is to build from scratch an agent that crawls citation networks, validates papers against a research topic, and synthesises a `related_works.tex` LaTeX snippet.

---

## Philosophy

We do not use any framework (like LangChain or DeepAgents), everything is done from first principles (without being too extreme).

---

## Architecture overview

The agent operates over a single mutable document: `state.json`.

```
state.json
* target_topic        The research question driving the search
* seed_keywords       Initial keywords for first API queries
* curated_library     Dict[paper_id, verified metadata]
* discovery_queue     List of paper IDs yet to be screened
* blacklist           Paper IDs explicitly rejected
* narrative_outline   Structured outline generated
* iteration           Loop counter (used for checkpointing)
```

The agent loop is a state machine:

THINK: call LLM with system prompt + state
PARSE: extract tool call from LLM JSON output
VALIDATE: check tool exists; catch hallucinations  
ACT: execute tool, get observation
UPDATE: write observation back to context
CHECKPOINT: write state.json to disk
REPEAT until terminal condition

## Setup

### 1. Install dependencies

```bash
# Install uv once (if you don't have it yet)
curl -LsSf https://astral.sh/uv/install.sh | sh   # macOS / Linux
# or: pip install uv

# Create the virtual environment and install all dependencies
uv sync

# Activate (optional - uv run works without it)
source .venv/bin/activate
```

`uv sync` reads `pyproject.toml`, resolves all dependencies, pins them in `uv.lock`, and writes the `.venv`. Re-run after any change to `pyproject.toml`.

To add a new package during the workshop:

```bash
uv add some-package
```

To run a single exercise without activating the venv:

```bash
uv run python day1/01_schema_parser.py
```

### 2. Configure your LLM backend

Copy `.env.example` to `.env` and fill in your credentials.

**Option A: Hugging Face Inference API (default)**

```bash
LLM_BACKEND=huggingface
HF_TOKEN=hf_...
HF_MODEL=Qwen/Qwen2.5-72B-Instruct
```

Free tier works. Models with reliable JSON output: `Qwen/Qwen2.5-72B-Instruct`, `meta-llama/Meta-Llama-3.1-8B-Instruct`, `mistralai/Mistral-7B-Instruct-v0.3`.

**Option B: LiteLLM (any provider)**

```bash
LLM_BACKEND=litellm
LITELLM_MODEL=gpt-4o-mini       # or anthropic/claude-haiku-4-5, etc.
OPENAI_API_KEY=sk-...           # whichever key your model needs
```

LiteLLM routes to OpenAI, Anthropic, Groq, Ollama, and 100+ others with the same interface.

### 3. Set your research topic

Edit `state.json`. Change `target_topic` and `seed_keywords` to match your own dissertation area. Everything else starts empty.

## Running the full agent

```bash
uv run python agent/main.py
uv run python agent/main.py --reset --topic "Your dissertation topic"
```

The agent will:
1. Load `../state.json`
2. Search for papers using the seed keywords
3. Evaluate each paper against the target topic
4. Traverse the citation graph of validated papers
5. Checkpoint state after every loop iteration
6. Pause for your input on uncertain papers
7. Cluster the curated library into themes
8. Generate `related_works.tex` and `references.bib`
9. Run a rigor check on the draft
